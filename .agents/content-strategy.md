# Content Strategy

Updated: 2026-07-20

## Pillars

1. **AI agent architecture** - context design, retrieval, tool use, evaluation, and token efficiency.
2. **Cloud data platforms** - governed pipelines, lakehouse patterns, reliability, and production economics.
3. **Public-data systems** - making official records searchable, explainable, and useful.

## Publishing Loop

1. Publish the durable version on `walidelkhoukh.com/notes/`.
2. Share the practical takeaway on LinkedIn for the professional audience.
3. Share the sharpest result or visual on X for the builder audience.
4. Cross-post to Medium and set the original article URL to the portfolio version.

Every piece should answer one concrete question, show one credible proof point, and offer one useful next step.

## Distribution Links

Use consistent UTM tags when sharing portfolio pages:

- LinkedIn: `utm_source=linkedin&utm_medium=social&utm_campaign=portfolio_notes`
- X: `utm_source=x&utm_medium=social&utm_campaign=portfolio_notes`
- Medium: `utm_source=medium&utm_medium=referral&utm_campaign=portfolio_notes`

## Measurement

Review monthly:

- Google Search Console: indexed pages, queries, click-through rate.
- Bing Webmaster Tools: indexing and search visibility.
- Vercel Web Analytics: visits and top pages.
- Microsoft Clarity: Perso/Pro usage, project clicks, note opens, CV downloads, and contact intent.

## Current Boundaries

- No paid acquisition.
- No newsletter until there is a repeat publishing rhythm.
- No interruptive popups.
- No A/B testing until traffic is large enough to produce useful evidence.
- Slide-Hustle remains supporting proof and an article example, not a dedicated case-study page.
